Changelog

What's changed in Morph

The three most recent releases. Morph is alpha — the on-disk format and CLI surface still move. Full history lives in CHANGELOG.md in the repo, and the unabridged record is the git log itself.

0.37.7 Latest 2026-04-29

Every Store::list(t) on FsStore is now a directory read instead of a whole-store walk. The latent hang that bit certify, then status + eval gaps, is closed at the source: every top-level object type finally has a per-type index dir.

Fixed

  • Type indexes for Blob, Tree, Pipeline, Commit, Artifact, TraceRollup. These were the last unindexed top-level types and the last latent versions of the same hang we'd patched twice already — anything that ever called list() walked the whole object fanout and JSON-deserialized every object just to filter by type. Each type now has a dedicated <type>/ index dir maintained by Store::put. Newly indexed types use zero-byte marker files so a 33 GB blob-heavy store doesn't double when blobs/ materializes; the five legacy indexes (runs/, traces/, evals/, prompts/, annotations/) keep their full-content format because at least one fallback path (morph tap) reads from them. Legacy stores rebuild every missing index in a single fanout pass on first use, so whichever list call happens first amortizes the cost for all subsequent list calls of every type.

Tests

  • Five new unit tests pin the contract end-to-end: every type uses the index (no get() calls), Pipeline gets its own fixture, legacy stores rebuild every missing index in one walk, the new types use zero-byte markers while the legacy types keep full content, and a prompt blob lands in both blobs/ and prompts/. Two read_dir consumers (activity_summary's counter and the cucumber run-record assertion) now filter on *.json so the .indexed marker doesn't inflate counts.
0.37.6 2026-04-29

Homepage now leads with a copy-able brew tap r/morph / brew install morph block in the hero, so macOS visitors can go from "land on the page" to "actually trying Morph" in two commands. Source-build path moves to a clearly-labelled fallback.

Changed

  • Hero install block on site/index.html. A new copy-able terminal panel sits above the CTA buttons with brew tap r/morph and brew install morph, plus a click-to-copy button and a footnote linking to the source-build path for non-macOS users. The #install section was rewritten to lead with the Homebrew path; cargo install --path … is kept as the cross-platform fallback rather than the only option.

CI

  • Releases auto-tag from Cargo.toml. A new .github/workflows/auto-tag.yml watches the workspace version on main; when it advances to a value that has not yet been tagged on origin, the workflow creates vX.Y.Z (annotated) and dispatches release-homebrew.yml against that tag. Cutting a release is now "bump Cargo.toml, push to main" — no more manual git tag && git push origin <tag>.
0.37.5 2026-04-29

morph status and morph eval gaps no longer hang on stores with many objects. Annotations finally get a per-type index dir, so listing them is a directory read instead of a full store walk.

Fixed

  • morph status and morph eval gaps got the same treatment as certify. Both call list_stale_certifications, which calls Store::list(ObjectType::Annotation). Annotations were the only top-level object type without a per-type index dir, so on FsStore the call walked the entire object fanout and JSON-deserialized every object just to filter by type — about two minutes per invocation on a 94k-object / 33 GB store. Annotations now have an annotations/ index dir maintained by put; legacy stores trigger a one-shot lazy rebuild on first use and drop an .indexed marker so subsequent calls hit the fast path. Verified on the bug-surfacing repo: first call pays the ~2 min rebuild once; second call 0.18 s (~600× speedup), morph status 0.17 s.

Tests

  • Three new unit tests pin the contract: legacy stores rebuild the index on first list(Annotation); once the marker is in place, even a poison file in the object fanout can't perturb the result; the rebuild correctly filters annotations away from noise blobs. New acceptance suite annotations_indexed.yaml (3 cases) covers certify populating the index, status rendering cleanly, and eval gaps succeeding on a certified branch.
