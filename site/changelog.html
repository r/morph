Changelog

What's changed in Morph

The three most recent releases. Morph is alpha — the on-disk format and CLI surface still move. Full history lives in CHANGELOG.md in the repo, and the unabridged record is the git log itself.

0.37.3 Latest 2026-04-29

morph eval gaps now reports stale certifications as a structured gap, so agents and CI can detect rebase-rotted evidence programmatically — not just from the human-readable morph status line.

Fixed

  • morph eval gaps surfaces stale certifications. When a morph commit was certified and a subsequent git commit --amend or git rebase attached a kind: "rewritten" annotation, morph status printed a one-line stale certification: N summary but the structured morph eval gaps --json output was silent. The new stale_certifications gap entry carries kind, count, a commits array of affected morph hashes, and a hint pointing at morph certify --commit <successor> --metrics .... Mode-agnostic — fires in standalone repos too whenever both annotation kinds end up on the same commit.

Tests

  • New acceptance suite eval_gaps_stale_certifications.yaml (3 cases) covering the reference-mode amend path, the audit-trail invariant that re-certifying the successor does not retroactively clear the gap, and the standalone-mode equivalent.

0.37.1 2026-04-28

Two reference-mode bugs that compounded each other: multi-commit fast-forward pulls were collapsing into one Morph mirror, and drift detection was hiding it.

Fixed

  • Reference mode, multi-commit fast-forward pulls. sync_to_head used to mirror only the tip of a multi-commit git pull, collapsing N new commits into a single Morph mirror with the wrong parent edge. It now walks first-parent ancestry back to the last-mirrored commit (or root) and mirrors the unmirrored span in topo-forward order via a shared sync_range helper that backfill_from_init also uses.
  • Reference mode, drift detection. drift_summary returned unmirrored_count = 0 whenever the git HEAD itself was mirrored, silently masking the multi-commit-pull bug above. The early-return is gone; the walk is uniform from HEAD until either a mirrored ancestor or a root, with the 10k-commit cap preserved.

Tests

  • New acceptance suites: reference_mode_multi_commit_pull.yaml (4 cases) and reference_mode_drift_topo.yaml (3 cases). Workspace: 1119 / 1119 passing.
