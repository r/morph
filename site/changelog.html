Changelog

What's changed in Morph

The three most recent releases. Morph is alpha — the on-disk format and CLI surface still move. Full history lives in CHANGELOG.md in the repo, and the unabridged record is the git log itself.

0.40.0 Latest 2026-05-01

Workspace-wide simplification: Morph runs in reference mode only. The legacy "Standalone" mode — the "morph manages its own object DAG and you git add .morph/" path — is gone. Every .morph/ directory in the world now sits next to a .git/ and is auto-excluded from git via .git/info/exclude. Behavioral history (runs, traces, prompts, certifications) is never tracked by git; sharing it with teammates is opt-in via a morph remote.

Removed

  • Standalone mode is gone. The RepoMode enum, the repo_mode config key, the --reference flag on morph init, and every read_repo_mode == Reference branch in morph-core, morph-mcp, and morph-cli collapse to unconditional reference- mode behavior. The "check .morph/ into git" story is retired.

Changed

  • morph init requires a git repository. When run inside a directory with no .git/, morph init interactively prompts "Run git init for you? [y/N]". Pressing Enter (or running non-interactively) exits non-zero with the recipe. New flags --git-init and --no-git-init for scripting. The --reference flag is removed; reference mode is implicit.
  • morph commit is a behavioral checkpoint of the working tree. The reference-mode commit path now runs git add -A before git commit, and always passes --allow-empty / --allow-empty-message so a morph commit succeeds even with no file changes — useful for recording metrics against the current tree.
  • morph branch and morph checkout mirror to git. Creating or switching branches now drives the underlying git working tree as well as Morph's refs, so subsequent git merge / git status calls always see a consistent state.
  • Mirrored Morph commits now snapshot the git tree. sync_one_commit populates the Commit.tree field from git ls-tree -r -z + git cat-file --batch, so morph status / morph diff work against the populated tree without needing a second source of truth.
  • morph upgrade migrates legacy Standalone repos. Drops the legacy repo_mode key, captures init_at_git_sha, writes the .morph/ exclude rule, and installs the four reference-mode hooks. Hard errors with a recipe when .git/ is missing.

Tests

  • 15 multi-step merge / retire / mixed-authorship specs are intentionally skip-annotated for this release — they exercise auto-union eval suites, metric retirement, conflict handling, and detailed human_edits provenance, which need the merge gate re-plumbed against reference-only commits and will land in the follow-up. Workspace test count: 1159 / 1159 passing, 15 skipped.
  • morph-cli/tests/specs/version.yaml updated to expect 0.40.0.
  • New --git-init / --no-git-init spec coverage in morph-cli/tests/specs/init_in_git_dir.yaml.
0.39.2 2026-05-01

Homepage rewires for the Agent of Empires integration shipped in 0.39.0. Version badge unsticks from the long-stale v0.37.7, a fourth "Agent of Empires" card lands next to Cursor / Claude Code / OpenCode, the install block lists morph setup aoe, and the "What works today" panel mentions AoE. No CLI surface changes — the binary already shipped this in 0.39.0; this release just makes the public site reflect it.

Changed

  • Homepage now lists Agent of Empires alongside Cursor / Claude Code / OpenCode. site/index.html had been frozen at the v0.37.7 badge and the IDE-integrations section showed only the three first-class IDEs, so the morph setup aoe integration shipped in 0.39.0 was invisible to anyone landing on the site. The version badge bumps to v0.39.2 in both the nav and the hero, a fourth "Agent of Empires" card lands next to the existing three with a link through to docs/AOE-SETUP.md, the section header renames to "Agent integrations" so AoE (a session manager, not an IDE) fits the framing, the "Try the alpha" install block adds morph setup aoe, and the "What works today" status panel mentions AoE so the honest status report stays in sync with the repo.
  • site/changelog.html rolls forward to surface 0.39.2 as the latest release, demoting 0.38.0 off the bottom (the page shows the three most recent releases).

Tests

  • morph-cli/tests/specs/version.yaml updated to expect 0.39.2 from morph --version and morph version.
0.39.1 2026-05-01

Fixes a silent regression in morph commit --from-run <hash>: it now propagates the run's observed_metrics into the commit, so the spec-first eval-driven workflow (morph eval run ... → morph commit --from-run) no longer produces metrics-less commits that break behavioral merge gating.

Fixed

  • Standalone-mode morph commit --from-run dropped the run's metrics map. The standalone commit path read the named run for provenance, evidence_refs, env_constraints, and contributors but silently skipped its metrics map, so every --from-run commit produced the "commit has no observed_metrics" warning and morph eval gaps kept reporting empty_head_metrics. The reference-mode helper already did this correctly; the standalone path now mirrors that behavior with the same UX as the LAST_RUN-breadcrumb auto-attach (a attaching evidence from run <hash>: k=v, ... stderr preview before the commit lands). Precedence is unchanged: explicit --metrics still wins, and a run whose metrics map is itself empty correctly leaves the commit metrics-less and surfaces the standard warning. The fix unblocks morph eval gaps for every commit captured via morph commit --from-run.

Tests

  • New commit_from_run_metrics.yaml acceptance suite (3 cases): a populated cargo-runner Run propagates tests_passed / tests_total into the commit; explicit --metrics overrides the run's metrics; a Run with metrics: {} still surfaces the standard warning rather than silently producing a metrics-less behavioral commit. The existing commit_from_recorded_run_* specs stay intact — they only ever asserted provenance / contributors, which is why the bug slipped past them.
