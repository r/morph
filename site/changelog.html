Changelog

What's changed in Morph

The three most recent releases. Morph is alpha — the on-disk format and CLI surface still move. Full history lives in CHANGELOG.md in the repo, and the unabridged record is the git log itself.

0.46.0 Latest 2026-05-02

Vocabulary unification, part 1. Fourth release of the “make morph intuitive” effort. Introduce morph session as the user-facing namespace for what's stored internally as a Run+Trace pair, and flatten the morph eval surface so cases and suites stop leaking from the storage model into the CLI. The old morph run *, morph eval add-case, morph eval suite-show, and morph eval suite-from-specs spellings continue to work in v0.46 and v0.47 with a one-line stderr deprecation notice; they're removed in v0.48. No on-disk format changes.

Added

  • morph session namespace subsumes morph run list, morph run show, morph run record-session, and the morph inspect export path under one coherent noun. Subcommands: list [--json], show <hash> [--with-trace] [--json], record [--prompt P --response R | --messages JSON], export [--mode ...]. Spec coverage: session.yaml.
  • Flat morph eval surface drops the Suite/Case noun split. New subcommands: eval add <files> (was add-case), eval show [--suite H] [--json] (was suite-show), eval rebuild <dirs> (was suite-from-specs). Identical semantics and identical effect on policy.default_eval_suite. Spec coverage: eval_flat.yaml.
  • New spec session_eval_deprecated_aliases.yaml pins the wording of the git-style stderr deprecation notice on every old command.

Changed

  • morph --help SESSIONS group replaces the v0.45 INSPECT group, listing session, inspect, show, head, identify, refs, annotate, annotations. The deprecated morph run namespace is hidden from --help.
  • Doc, site, and asset sweep. Every morph run record-session / morph run list / morph eval add-case / morph eval suite-show / morph eval suite-from-specs mention in README.md, docs/EVAL-DRIVEN.md, docs/v0-spec.md, docs/SECURITY.md, docs/SESSION-TRACKING.md, the IDE setup guides (AOE-SETUP, OPENCODE-SETUP, CURSOR-SETUP), the cursor / opencode rule files, the opencode plugin script (morph-record.ts), and site/index.html rewritten to the new spellings. Freshly-installed opencode hooks now call morph session record directly.

Deprecated

  • morph run {list, show, record-session} — use the corresponding morph session subcommand. Removed in v0.48.
  • morph eval {add-case, suite-show, suite-from-specs} — use morph eval {add, show, rebuild}. Removed in v0.48. Each invocation prints a one-line git-style stderr notice (warning: `morph eval add-case` is deprecated; use `morph eval add` instead (removed in v0.48).).

Tests

  • Workspace 1244 / 1244 passing (1226 baseline + 18 new acceptance cases). Cucumber 34 / 37 (3 skipped, 0 failed) — no change.
0.45.0 2026-05-02

Inspection collapse. Third release of the “make morph intuitive” effort. Three near-synonymous CLIs (morph trace, morph tap, morph traces) become one coherent morph inspect namespace with twelve subcommands — summary, recent, show, diagnose, export, stats, preview, task, target, artifact, semantics, verification. The new inspect show <hash> auto-detects whether the hash points at a Trace (raw events) or a Run (structured task), collapsing two old commands into one. The old commands continue to work in v0.45 and v0.46 with a one-line stderr deprecation notice; they're removed in v0.47.

Added

  • morph inspect namespace with twelve subcommands consolidating the older morph trace, morph tap, and morph traces. New module morph-cli/src/inspect.rs factors the handlers out of main.rs so the new namespace and the deprecated aliases share one set of entry points. Spec coverage: inspect.yaml, inspect_deprecated_aliases.yaml.
  • morph inspect show <hash> auto-detects Trace vs Run on the hash so users don't have to remember which old subcommand applied. all still iterates every Run.

Changed

  • morph --help INSPECT group leads with inspect; the deprecated trace, tap, traces are hidden from the help output (they still work on the command line) so new readers see the right spelling first.
  • Doc + site sweep. Every morph tap / morph trace / morph traces mention in README.md, docs/v0-spec.md, docs/SECURITY.md, docs/SESSION-TRACKING.md, and the three public site pages (index.html, tutorials/getting-started.html, tutorials/adding-to-git-project.html) rewritten to use morph inspect .... Architectural prose (“morph traces live in the commit graph”) is left alone.

Deprecated

  • morph trace show, morph tap {summary, inspect, diagnose, export, trace-stats, preview}, morph traces {summary, task-structure, target-context, final-artifact, semantics, verification} — use the corresponding morph inspect subcommand. Each invocation prints a one-line git-style stderr notice (warning: `morph tap summary` is deprecated; use `morph inspect summary` instead (removed in v0.47).). Removed in v0.47.

Tests

  • Workspace 1226 / 1226 passing (1203 baseline + 22 new acceptance cases plus one extra registration). Cucumber 34 / 37 (3 skipped, 0 failed) — no change.
0.44.0 2026-05-02

Commit ergonomics. Second release of the “make morph intuitive” effort. morph commit becomes a one-liner even when you want metrics: tell Morph your test suite once with morph config commit.test_command "cargo test --workspace" (or pytest, vitest, jest, go) and from then on plain morph commit shells out to the command, parses the output, writes a fresh LAST_RUN.json breadcrumb, and attaches the metrics + run as evidence — collapsing the eval-driven flow from two commands to one for the common case. --no-test opts out for a single commit, --rerun forces a fresh run when the breadcrumb is stale, and a failing test aborts the commit so red suites don't slip into the behavioral history.

Added

  • commit.test_command config key in .morph/config.json. Set with morph config commit.test_command "…"; read with morph config commit.test_command or --get; exit 1 when unset (matches the user.* convention). The string is POSIX-shell-split with shlex so quoted arguments survive ("pytest -k 'fast and not slow'"). Spec coverage: commit_runs_configured_test_command.yaml, config_commit_test_command.yaml.
  • morph commit --no-test — skip the configured auto-run for this commit without removing the configured command. Useful for chores or out-of-band certification.
  • morph commit --rerun — force a fresh test run even when the most recent breadcrumb is current. Use after an external state change makes cached metrics stale.

Changed

  • Failing tests abort the commit. A non-zero exit from commit.test_command is treated as evidence the code is not in a committable state. The failing run is still recorded (inspect with morph show <hash>), but the commit doesn't land. Override with --no-test.
  • README quickstart leads with the configured command. The recommended one-time setup is morph config commit.test_command "…"; the old two-step (morph eval run then morph commit) is preserved as the manual fallback for repos that prefer it.
  • docs/EVAL-DRIVEN.md §5 features the one-command flow as the primary path. The breadcrumb-pickup and explicit --from-run <hash> forms are demoted to subsections.
  • Tutorials updated. site/tutorials/getting-started.html §7 gains a "Skip the two-step?" callout pointing at the configured-command flow; site/tutorials/adding-to-git-project.html rewrites the post-staging commit example to use it.

Tests

  • Workspace 1203 / 1203 passing (1192 baseline + 6 new acceptance cases across the two new spec files + 5 new unit tests in morph-core/src/commit_config.rs). Cucumber 34 / 37 (3 skipped, 0 failed) — no change.
