Changelog

What's changed in Morph

The three most recent releases. Morph is alpha — the on-disk format and CLI surface still move. Full history lives in CHANGELOG.md in the repo, and the unabridged record is the git log itself.

0.41.1 Latest 2026-05-01

Reference-mode merge UX gets cleaned up. Three fixes triaged out of the v0.40 ignored-spec backlog: morph status now shows an in-progress git merge conflict (it was reading the legacy structural-merge state file and missing the new MERGE_REF.json breadcrumb); morph merge --abort outside a merge errors with exit 1 instead of no-op'ing on exit 0, matching git merge --abort's own UX; and the conflict message ("Auto-merging failed for N path; run morph merge --continue or --abort") is now exercised by an un-skipped acceptance spec. 13 ignored specs remain — all sharing two clear roots (eval-suite plumbing through the ref-mode merge rebuild path, mixed-authorship plumbing on the rebuild path) — and are scheduled for v0.42.0 / v0.42.1.

Fixed

  • merge_progress_summary in morph-core/src/merge_flow.rs reads .morph/MERGE_REF.json in addition to the legacy .morph/MERGE_HEAD, so morph status mid-conflict reports "You have unmerged paths" with the --continue / --abort hint banner. Unmerged paths come from git ls-files --unmerged.
  • morph merge --abort outside a merge now bails with Error: no merge in progress (exit 1) instead of printing "Nothing to abort." and returning success. Mirrors git merge --abort's fatal: There is no merge to abort (exit 128). Earlier ref-mode versions were silent in scripts when the user mistyped a command they expected to fail.

Tests

  • merge_with_textual_conflict_drops_into_continue_flow and merge_abort_without_in_progress_errors in morph-cli/tests/specs/merge.yaml un-skipped and rewritten for v0.40+ wording (morph's hints are on stderr now since git merge -q swallows git's own CONFLICT (content): ... lines). status_during_textual_merge_lists_unmerged_paths_and_hint in morph-cli/tests/status_merge_integration.rs un-#[ignore]'d. version.yaml updated to expect 0.41.1.
0.41.0 2026-05-01

morph forget lands. The "I leaked a secret into a trace; what do I do?" question now has a first-class answer: a CLI subcommand that permanently retires a Run, Trace, or prompt Blob from the local store and (with --remote <name>) propagates the deletion to a configured morph remote via an immutable, content-addressed Tombstone object. Teammates' next morph fetch applies the tombstone automatically. The merge gate reads any evidence_ref that resolves to a tombstone as "no claim" rather than a hard error, so retroactively forgetting evidence does not retroactively break commits. Forget refuses commits, trees, regular blobs, pipelines, eval suites, artifacts, trace rollups, and annotations — those carry structural meaning the DAG depends on.

Added

  • morph forget <hash> CLI subcommand. Flags: --reason, --force, --remote, --dry-run, --yes (required for non-interactive callers; interactive callers type forget to confirm). Writes the tombstone, deletes the original object + per-type index entries, and drops a .morph/forgotten/<hash>.txt marker.
  • Tombstone as a first-class MorphObject. Fields: original_hash, original_kind, forgotten_at, actor, reason. Content-addressed, so morph show <tombstone-hash> returns who retired what, when, and why. The original bytes are gone; the fact of the deletion is permanent.
  • Push / fetch propagation. morph push ships tombstones alongside normal objects on local-filesystem morph remotes; morph fetch auto-applies them on receipt. SSH transport for tombstones is queued for v0.41.1.
  • docs/SECURITY.md — full morph forget section replaces the v0.40.2 gap-bullet: when to use it, how it works, what it refuses, what it does not cover (already-fetched copies, partial redaction, SSH propagation, MCP tool), and a step-by-step "I leaked a secret" recipe.
  • Homepage morph forget callout. site/index.html privacy section now leads with the secret-leak escape hatch: one paragraph, a three-line CLI sample, deep-link to SECURITY.md, and a "what forget does not cover" footer. The "Things morph does not yet do" list drops the missing-forget bullet.

Tests

  • New morph-core::forget unit tests pin the local round-trip, apply-tombstone idempotency, kind refusals, and referencing-commit refusals. morph-cli/tests/specs/forget.yaml registers six CLI cases through the auto-generated harness (forget a run with --yes, --dry-run is read-only, non-interactive without --yes is refused, commit-kind hashes are refused, runs referenced by a commit are refused without --force, the original is no longer loadable post-forget). The cross-repo push/fetch propagation case is covered by the morph-core unit tests; an end-to-end Cucumber feature for the multi-repo CLI flow is queued for v0.41.1. morph-cli/tests/specs/version.yaml updated to expect 0.41.0.
0.40.2 2026-05-01

Docs-only release: the plain privacy & sharing story. New docs/SECURITY.md answers the question we keep getting asked — "morph records everything; what happens when I share it?" — without hand-waving. It states up front what a trace contains (prompts, responses, file contents the agent read, shell stdout/stderr, env), where it lives (.morph/, never tracked by git, not encrypted at rest), what crosses the wire when (a two-channel diagram: git push can't carry traces, morph push is opt-in and sends everything reachable), the team-setup checklist, and the gaps morph does not yet have — including no morph forget, which is the headline feature of the next release line (v0.41.0). The homepage gets a compact "What gets recorded, what gets shared" section between How-It-Works and Design-Principles, mirroring the same two-channel framing.

Added

  • docs/SECURITY.md. New, plain-language companion to docs/SESSION-TRACKING.md: what morph records said plainly, the on-disk layout (and the fact that .morph/ is never git-tracked), git push vs morph push drawn as a two-channel diagram, the recommended team setup, the checklist before you share, the known-gaps list (no morph forget, no client-side redaction filter on push, no selective fetch, no encryption at rest, no automatic secret scanning), and a brittle-recipe for "if you leak a secret into a trace, what do you do today?" that collapses into a one-liner once morph forget ships in v0.41.0.
  • Homepage privacy section. site/index.html — new "What gets recorded, what gets shared" section between How-It- Works and Design-Principles. Two side-by-side cards (git push: code only; morph push: opt-in, separate) make the two-channel model visible at a glance, followed by a "things morph does not yet do" list and a deep-link to the full SECURITY.md. Adds a .privacy-grid + .privacy-card + .privacy-list style block to the page's CSS.
  • docs/README.md Guides table indexes SECURITY.md next to SESSION-TRACKING.md, with the "read before you push to a morph remote" hint.

Tests

  • morph-cli/tests/specs/version.yaml updated to expect 0.40.2. Workspace test count unchanged from 0.40.1: 1159 / 1159 passing, 15 skipped.
0.40.1 2026-05-01

Docs-only release: a clean, plainly-argued session-tracking story. New docs/SESSION-TRACKING.md opens with the load-bearing claim — "a diff plus a commit message is not enough to review AI-authored code; the next person needs the prompt" — and lays out seven concrete things you can only do because the prompt + trace are part of the commit graph (review the transformation, replay, attribution, prompt-as-spec, prompts → acceptance cases, merge-aware behavioral context, cross-tool portability). It then puts a six-row comparison table against the alternatives we keep getting asked about — Claude / Cursor / OpenCode on-disk transcripts, OTEL hooks, and Langfuse / Phoenix / Helicone — with the specific gap each one has: not linked to a commit, not content-addressed, not merge-aware, not local-first. The homepage gets the same comparison row embedded inside the existing Runs and Traces section, and a deep-link out to the full doc.

Added

  • docs/SESSION-TRACKING.md. New, ~330-line companion to docs/MORPH-AND-GIT.md: why a diff is not enough for AI-authored code, the seven first-class things morph traces unlock, why the alternatives (Claude on-disk transcripts, OTEL spans in a tracing backend, Langfuse / Phoenix / Helicone) cannot deliver what the morph trace contract delivers, and the tool-agnostic Run + Trace contract itself. Forward-links to docs/SECURITY.md (landing in 0.40.2) for the privacy and team-sharing follow-up.
  • Homepage comparison table. site/index.html — the existing "Runs and Traces" solution item now ends with a 6-row comparison: linked to a commit, content- addressed, visible to teammates, same shape across tools, merge-aware, local-first. Each row scores Claude/Cursor/OpenCode transcripts, the Langfuse/Phoenix/OTEL world, and morph honestly — "yes morph is local-first; no Langfuse isn't; yes Cursor's logs live on your laptop too but they aren't linked to commits." Links out to the full doc on GitHub.
  • docs/README.md index entry for SESSION-TRACKING.md in the Guides table.

Tests

  • morph-cli/tests/specs/version.yaml updated to expect 0.40.1. Workspace test count unchanged from 0.40.0: 1159 / 1159 passing, 15 skipped.
